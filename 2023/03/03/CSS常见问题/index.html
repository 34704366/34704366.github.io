CSS常见问题

CSS常见问题

  1. 垂直居中方式

    1. position:absolute ，设置left、top、margin-left、margin-top等属性调整，这种方式是浏览器都能兼容，设计上较为简单，缺点是不够灵活，需要固定的长宽。

    2. position:fixed IE不支持

    3. 设置line-height使行内元素居中，文字垂直居中较为常用

    4. CSS3 transform:**translate(x,y)**，不需要设置固定的宽高，移动端用的较多

    5. 设置一个:before的伪类，在前面用100%高度撑起整个盒模型

    6. table布局，设置父元素为display:table-cell，子元素设置 display: inline-block。利用vertical-align和text-align可以让所有的行内块级元素水平垂直居中

    7. flex布局

    8. 设置CSS3的-webkit-box属性

  2. PPI（每英寸像素）和DPR（设备像素比）DIP（设备独立像素）

    DPR = 物理像素 / DIP(虚拟)

    可以通过window.devicePixelRatio获取

  3. CSS隐藏元素

    1. dispaly:none，设置后该元素从render tree中消失（但是还在DOM tree中），不占据页面空间，会导致一次重排和重绘，元素不可见也无法触发事件

    2. visibility:hidden

      仅仅是隐藏，占据页面空间但也不会触发点击事件，不会触发重排，会触发重绘

    3. opacity:0

      可以响应点击事件

    4. position:absolute

      元素不可见，不影响页面布局，但是还是存在于render tree和layout tree中

  4. BFC（Block Formatting Context，块级格式化上下文）

    特性：

    • 同一个BFC中两个相邻的盒子的margin会重叠（以最大的为准）
    • 每个元素的左外边距与包含块的左边界相接触（从左到右），即使浮动元素也是如此
    • BFC的区域不会与float的元素区域重叠
    • 计算BFC的高度时，浮动子元素也参与计算
    • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然

    常用overflow: hidden触发形成一个单独的BFC（可以用来消除浮动）

  5. 三栏布局

    1. flex布局实现（自适应）两边设置固定的宽度，中间用width100%或者设置flex:1

    2. 两边用float，中间用margin根据两边的宽度定义（适应性较差）

  6. CSS3有什么新属性

    1. border-radius / border-image

    2. word-wrap: normal|break-word ,break-word允许在单词内换行

    3. text-shadow / box-shadow

    4. text-decoration

    5. rgba() hsla(),（h色相，s饱和度，l亮度，a透明度）

    6. transition过渡动画

    7. transform转换动画

      • transform: translate(120px, 50%)：位移
      • transform: scale(2, 0.5)：缩放
      • transform: rotate(0.5turn)：旋转
      • transform: skew(30deg, 20deg)：倾斜

    8. animation自定义动画

    9. linear-gradient / radial-gradinet

    10. flex弹性布局、grid栅格布局

  7. asd

  8. asd

  9. asd
